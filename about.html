Scope

ARC-20 is a UTXO coloring model.

One ARC-20 unit maps to one satoshi. A recipient controls units only when Atomicals allocation rules validly assign the colored quantity to a Bitcoin UTXO they control.

What this documentation is

A field guide for understanding the protocol, validating implementation ideas, tracing source-dependent rules, and separating the current Universe product scope from protocol capabilities.

What it is not

Not an ERC-20 account-balance protocol, price guarantee, redemption mechanism, investment thesis, validator substitute, or official Atomicals protocol site.

Accuracy policy

Useful claims have an explicit boundary.

The Atomicals specification guide says the specification is defined in code. This site therefore treats human-readable protocol material and active validator behavior as different evidence layers.

  • Explanatory statements link to Atomicals guide material.
  • Source-dependent behavior is pinned to an ElectrumX revision in the source ledger.
  • Protocol capability is separated from current Universe feature availability.
  • Activation-gated behavior is labeled instead of presented as universal support.
  • Metadata, tickers, and generic confirmations are not presented as authenticity proof.

Important product boundary

Do not sell a protocol feature as a current product feature.

The site documents the local ARC-20 product path as an audited snapshot, not a roadmap or public API stability promise.

Current safety note

Direct mint-ft issuance and the visible ARC-20 transfer path must not be described as supported today. The visible transfer flow does not supply the colored outpoints its backend builder requires, and is not partial-transfer, split, or combine support.
