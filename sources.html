Update process

Change sources and documentation together.

Protocol-facing language must remain pinned, traceable, and testable. Avoid silently expanding a product claim from an explanatory source or inactive code path.

  • Select and record an ElectrumX release or commit.
  • Check the active network and activation conditions relevant to the behavior.
  • Re-run behavioral vectors for direct FT, fixed DFT, DMT, normal allocation, burn handling, split, and custom coloring as applicable.
  • Update the HTML specification, reference, guide, machine-readable file, and source ledger together.
  • Do not turn a metadata convention, third-party feature, or inactive code path into a protocol-wide claim.
