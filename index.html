Bitcoin, Atomicals, colored satoshis

ARC-20
mapped.

A field guide to fungible Atomicals units on Bitcoin. ARC-20 is a UTXO coloring model: one token unit maps to one satoshi, and a valid token movement depends on the transaction outputs as well as the Bitcoin spend.

Read this first

The specification is code-defined.

The Atomicals guide explicitly directs implementers to the code for the specification. This site is a researched secondary guide, not an alternative validator. It pins implementation-dependent statements to a source revision and calls out places where a newer indexer can change the outcome.

Authority order

Use the Atomicals protocol guide for orientation, the active Atomicals ElectrumX validation code for rule behavior, then a compatible wallet or indexer for building and checking a particular transaction.

A confirmed Bitcoin transaction is not enough

A Bitcoin transaction can confirm while its ARC-20 allocation burns units or differs from the intended recipient split. Treat an indexer blueprint check and the post-broadcast allocation as part of the transaction result.

Mental model

Token balance is not an account field.

ARC-20 does not use an ERC-20 contract or a ledger balance stored in an account. A compatible reader interprets Atomicals history and transaction allocation rules to determine which spendable outputs carry a particular token's colored satoshis.

1 unitOne ARC-20 unit corresponds to one colored satoshi. Native quantities are integers.
txid i voutAn Atomical ID identifies a minted Atomical by transaction ID and output index, for example <txid>i0.

Identify the asset precisely

Use the Atomical ID, ticker resolution, mint mode, and history. A ticker, image, project name, or marketplace label does not independently authenticate an asset.

Identify protected inputs

Locate the colored outpoints and distinguish them from ordinary, or cardinal, BTC used to pay transaction fees.

Construct the intended allocation

Output order and output values affect normal coloring. Preview the exact allocation instead of trusting a generic send amount.

Verify the resulting outputs

After broadcast, verify the recognized destination outpoints, any burn result, and the current indexer or implementation version that evaluated them.

Issuance paths

Two core models, different trust surfaces.

ARC-20's official guide describes direct and decentralized issuance. They should never be treated as interchangeable when evaluating supply, issuer authority, or whether a particular product supports the transaction.

Protocol path A

Direct FT mint

The reference CLI form is mint-ft <tick> <total_supply> metadata.json. The full supply is placed in one output, where each satoshi represents a unit. This is a protocol capability, not a claim that every wallet or app offers it.

Use when

An issuer needs to create the complete supply at once and distribute it later from the initial colored output.

Protocol path B

Fixed decentralized mint

The guide's init-dft creates rules for a public mint. A fixed campaign has mint_amount, max_mints, and mint_height, with nominal maximum issuance mint_amount * max_mints.

Use when

Participants can claim individual valid mints after the configured start height, subject to the deployment's quota and any applicable Bitwork rule.

Perpetual DFT is separate

Current reference code also has activation-gated perpetual DFT behavior. Do not infer a fixed supply from the word "decentralized". Read the deployment parameters and the indexer version before displaying a cap.

Transfer safety

Output values can burn units.

Normal FT allocation walks eligible transaction outputs in order. Under the pinned reference behavior, an output is colored only while its full BTC value fits in the remaining colored amount. A too-large next output or exhausted output list can burn the remainder.

Do

Use a token-aware builder

Protect Atomicals outpoints from automatic coin selection, and fund miner fees with separate cardinal UTXOs whenever the builder requires it.

Check

Preview values and ordering

Inspect inputs, output positions, output sat values, token values, and any allocation or burn report before signing.

Avoid

Generic Bitcoin sends

A wallet that does not understand Atomicals can create a Bitcoin-valid spend that does not preserve the intended ARC-20 allocation.
